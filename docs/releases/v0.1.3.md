# Release Notes v0.1.3

Release date: 2026-03-20

## Summary

This release turns the scaffold into a more complete LangGraph starter for
OpenAI-compatible APIs. Workflow-local LLM ownership is now the default,
streaming behaves as true SSE, `demo-chat` demonstrates persisted multi-turn
conversation, and the public API / config surface has been cleaned up around
kebab-case workflow names and `sysCode` caller identity.

## Highlights

- Added `demo-chat` for multi-turn chat with both streaming and non-streaming
  responses.
- Moved upstream LLM ownership into workflow-local `llm.default` config.
- Tightened API auth around `api.auth.sys_codes`.
- Renamed external workflow identifiers to kebab-case:
  - `demo-chat`
  - `demo-hitl`
  - `demo-summary`
- Added configurable Nacos SDK logging through `dynamic-config-nacos 0.1.2`.

## What Changed

### Workflow Runtime

- `WorkflowRuntime` now scopes LangGraph `thread_id` by:
  - `workflow`
  - `sys_code`
  - `user-id`
  - `session-id`
- `demo-chat` uses LangGraph `MessagesState` plus the configured checkpointer
  to restore prior conversation turns.
- Request replay is handled safely: if callers resend already persisted
  history, the runtime avoids appending duplicate turns.

### API and Auth

- The request `model` field is mandatory and maps directly to a workflow.
- Caller identity now uses:
  - request header `sysCode`
  - internal field `sys_code`
- Auth is configured through:

```yaml
api:
  auth:
    enabled: true
    sys_codes:
      - demo-system
      - real-system
```

- Missing or unauthorized caller identity now returns `401 invalid_sys_code`.
- Caller business identity is no longer forwarded to upstream LLM requests.

### Workflow Naming

- Public workflow names changed from snake_case to kebab-case:
  - `demo_chat` -> `demo-chat`
  - `demo_hitl` -> `demo-hitl`
  - `demo_summary` -> `demo-summary`
- Python package directories remain unchanged in `src/workflows/` so imports and
  code organization stay stable.
- Workflow config files were renamed to:
  - `configs/workflows/demo-chat.yaml`
  - `configs/workflows/demo-hitl.yaml`
  - `configs/workflows/demo-summary.yaml`

### LLM Config

Workflow-local upstream LLM config is expected under:

```yaml
llm:
  default:
    provider: openai_compatible
    base_url: https://your-llm.example.com/v1
    apikey: your-token
    headers:
      X-Tenant: your-tenant
    model: your-model
    max_tokens: 2048
    timeout: 30000
    retry:
      attempts: 2
      min_wait: 200
      max_wait: 1000
```

### Nacos

- Upgraded `dynamic-config-nacos` to `0.1.2`.
- Added Nacos SDK log configuration support:

```yaml
nacos:
  sdk_log_path: logs/nacos.log
  sdk_log_level: INFO
```

- The same log settings can also be provided by:
  - `NACOS_SDK_LOG_PATH`
  - `NACOS_SDK_LOG_LEVEL`

### Demo Coverage

- `demo-summary`: non-HITL summary flow
- `demo-hitl`: interrupt/resume human-in-the-loop flow
- `demo-chat`: persisted multi-turn chat flow

## Upgrade Notes

- Re-sync dependencies after pulling the release:

```bash
uv sync --python 3.12 --all-extras
```

- Re-publish root and workflow config files to Nacos if you use the helper
  script:

```bash
bash scripts/push_nacos_configs.sh
```

- Restart the service after pulling so the updated workflow registry and config
  naming take effect:

```bash
uv run --python 3.12 uvicorn --env-file .env --app-dir src app.main:app --host 127.0.0.1 --port 8080
```
