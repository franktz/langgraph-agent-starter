# Release Notes v0.1.2

Release date: 2026-03-16

## Summary

This release turns the scaffold into a more complete workflow starter for
OpenAI-compatible APIs: workflow-local LLM config is now the default model
ownership boundary, true SSE streaming is in place, and a new `demo_chat`
workflow demonstrates persisted multi-turn conversation on top of LangGraph
checkpointers.

## Highlights

- Added `demo_chat` for multi-turn chat with both streaming and non-streaming
  responses.
- Moved LLM runtime ownership into workflow-local `llm.default` config.
- Tightened API auth around `api.auth.systemkeys`.
- Removed reliance on default workflow routing; `model` is now required.
- Improved persisted-thread isolation by combining workflow, system, user, and
  session identity.
- Refreshed README and cURL examples to cover `demo_chat` and current runtime
  behavior.

## What Changed

### Workflow Runtime

- `WorkflowRuntime` now binds LangGraph `thread_id` by:
  - `workflow`
  - `systemkey`
  - `user-id`
  - `session-id`
- `demo_chat` uses LangGraph `MessagesState` plus the configured checkpointer
  to restore prior conversation turns.
- Request replay is handled more safely: if callers resend already persisted
  history, the runtime avoids appending duplicate turns.

### API and Config

- The request `model` field is mandatory and maps directly to a workflow.
- `systemkey` is now strictly a caller/business identity header.
- Auth is configured through:

```yaml
api:
  auth:
    enabled: true
    systemkeys:
      - demo-system
      - real-system
```

- Workflow-local upstream LLM config is now expected under:

```yaml
llm:
  default:
    provider: openai_compatible
    base_url: https://your-llm.example.com/v1
    apikey: your-token
    headers:
      X-Tenant: your-tenant
    model: your-model
    max_tokens: 2048
    timeout: 30000
    retry:
      attempts: 2
      min_wait: 200
      max_wait: 1000
```

### Demo Coverage

- `demo_summary`: non-HITL summary flow
- `demo_hitl`: interrupt/resume human-in-the-loop flow
- `demo_chat`: persisted multi-turn chat flow
