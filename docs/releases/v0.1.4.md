# Release Notes v0.1.4

Release date: 2026-03-26

## Summary

This release expands persistence options, simplifies runtime state handling,
and improves non-conversation workflow inputs. Checkpointers can now be
disabled explicitly or backed by Redis, MySQL, or MongoDB, while
non-conversation workflows now receive both normalized and raw request messages
per request without relying on the legacy `question` field.

## Highlights

- Upgraded `dynamic-config-nacos` to `0.1.3`.
- Added MySQL and MongoDB checkpointer backends.
- Added Redis `cluster_mode` support for checkpointer configuration.
- Added `backend: null` support to disable the checkpointer cleanly.
- Added `input_messages` and `raw_input_messages` for non-conversation
  workflows.
- Replaced the legacy `question` workflow input with request-scoped message
  inputs plus the public `last_text_from_input_messages(...)` helper.

## What Changed

### Checkpointer Configuration

Checkpointer configuration is now grouped by backend type under
`langgraph.checkpointer`:

```yaml
langgraph:
  checkpointer:
    backend: redis
    redis:
      url: redis://localhost:6379/0
      cluster_mode: false
    mysql:
      host: 127.0.0.1
      port: 3306
      user: root
      password: secret
      database: langgraph
    mongodb:
      uri: mongodb://localhost:27017
      database: langgraph
```

Supported values for `backend` are:

- `null`
- `memory`
- `redis`
- `mysql`
- `mongodb`

When `backend: null`, LangGraph persistence is disabled entirely.

### Runtime Input Semantics

For workflows with `supports_conversation = false`, the runtime now injects:

- `input_messages`
  Schema-normalized request messages suitable for workflow logic.
- `raw_input_messages`
  Raw request body messages preserving undeclared fields from the caller.

This is especially useful for one-shot flows that still need access to the
incoming request message list even when a checkpointer is configured.

The runtime now overwrites persisted request-scoped fields on each request
instead of accumulating them:

- `input_messages`
- `raw_input_messages`
- `sys_code`

### Workflow Helpers

- Added `last_text_from_input_messages(messages, roles=None)`.
- By default it inspects the latest `tool` or `user` message with text content.
- It also supports a single role such as `"user"` or a collection such as
  `{"tool", "user"}`.

### Runtime Cleanup

- Removed `WorkflowRuntime._session_state`.
- Removed the legacy `question` field from non-conversation workflow state and
  node usage.

## Upgrade Notes

- Re-sync dependencies after pulling the release:

```bash
uv sync --python 3.12 --all-extras
```

- If you use Redis cluster mode for persistence, configure it explicitly under
  `langgraph.checkpointer.redis.cluster_mode`.

- If you previously read `state["question"]` in custom non-conversation
  workflows, migrate to `state["input_messages"]` and extract text from the
  latest relevant message instead.
