# Release Notes v0.1.1

Release date: 2026-03-15

## Summary

This release focuses on stabilizing the Nacos integration path used by the
starter project during local development and packaging.

## Highlights

- Upgraded `dynamic-config-nacos` to `0.1.1`.
- Pinned `nacos-sdk-python` to `2.0.11`.
- Bumped `langgraph-agent-starter` package version to `0.1.1`.
- Added a changelog for tracking future releases.
- Refreshed the English and Chinese README files and exposed release-document
  entry points.

## Why This Release

The previous dependency combination could surface Nacos SDK import mismatches
in local environments, especially on Windows, when the runtime expected the
`sdk_v2` style `import nacos` module path but resolved a `3.x` SDK instead.

Version `0.1.1` aligns the dependency set with the currently validated runtime
path and makes the release story clearer for downstream users.

## Upgrade Notes

- Re-sync dependencies after pulling the release:

```bash
uv sync --python 3.12 --all-extras
```

- If you start the app through `uvicorn`, prefer loading environment variables
  from `.env`:

```bash
uv run --python 3.12 uvicorn --env-file .env --app-dir src app.main:app --reload
```

## Validation

- Verified resolved package versions:
  - `langgraph-agent-starter==0.1.1`
  - `dynamic-config-nacos==0.1.1`
  - `nacos-sdk-python==2.0.11`
- Verified `uv build` completes successfully for both sdist and wheel.
